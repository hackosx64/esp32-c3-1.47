# ESP32-C6 Экран тестового генератора

Проект для PlatformIO/ESP-IDF, реализующий тестовый генератор сигналов для ESP32-C6 с управлением от энкодера и отображением выбранной формы и частоты на OLED-дисплее SSD1306 (горизонтальная ориентация). Выходная форма строится на канале LEDC, частота и форма сигнала выбираются поворотом и нажатием энкодера.

## Возможности

- Предустановленные формы: синус, треугольник, пила, меандр, импульсный ШИМ 25%, постоянный уровень, пачки импульсов и «ступеньки».
- Режим выбора формы (вращение энкодера) и редактирования частоты (первое нажатие — переход в режим, второе — сохранение значения и возврат к выбору формы).
- Горизонтальная шкала визуализации сигнала на дисплее и строка частоты внизу, при редактировании частоты значение мигает.
- Возможность быстро изменить шаг регулировки частоты в зависимости от диапазона (1/5/10/20 Гц) в пределах рабочего диапазона 1–780 Гц.

## Аппаратная схема

| Узел | GPIO ESP32-C6 | Примечание |
| --- | --- | --- |
| SSD1306 SDA | GPIO6 (изменяемо через макрос `SSD1306_SDA_GPIO`) | I²C, подтяжка 4,7 кОм к 3,3 В |
| SSD1306 SCL | GPIO7 (макрос `SSD1306_SCL_GPIO`) | I²C, подтяжка 4,7 кОм к 3,3 В |
| SSD1306 VCC | 3,3 В | |
| SSD1306 GND | GND | |
| Энкодер A | GPIO8 (макрос `ROTARY_ENCODER_A_GPIO`) | Включена программная подтяжка к 3,3 В |
| Энкодер B | GPIO9 (макрос `ROTARY_ENCODER_B_GPIO`) | |
| Энкодер кнопка | GPIO10 (макрос `ROTARY_ENCODER_BUTTON_GPIO`) | Кнопка замыкается на землю |
| Аналоговый выход | GPIO4 (макрос `WAVE_OUTPUT_GPIO`) | Сигнал PWM (фильтровать RC для получения «аналоговой» формы) |

> При необходимости вы можете переопределить GPIO через `build_flags` в `platformio.ini` или на уровне кода.

Для получения сглаженного сигнала подключите RC-фильтр (например, R=1 кОм, C=100 нФ) к выводу GPIO4. Если требуется более высокая амплитуда — используйте внешний усилитель/буфер.

## Подготовка окружения

1. Установите [PlatformIO](https://platformio.org/install) или используйте уже установленную среду с ESP-IDF.
2. Откройте папку проекта:

   ```bash
   cd esp32-c3-1.47
   ```

3. Проверьте файл `platformio.ini` и при необходимости подкорректируйте параметры (скорость монитора, номера GPIO и т. д.).
   В разделе `[platformio]` предустановлена среда `esp32c6`, поэтому при сборке в VS Code или из CLI автоматически выбирается правильная конфигурация.

## Сборка и прошивка

```bash
pio run
pio run --target upload
pio device monitor
```

При первой прошивке рекомендуется убедиться, что опция `CONFIG_LEDC_ISR_IRAM_SAFE` включена в меню `menuconfig` (необходимо для корректной работы обновления ШИМ из обработчика таймера). В PlatformIO это можно сделать командой:

```bash
pio run -t menuconfig
```

В разделе `Component config -> LEDC (LED Controller)` активируйте `Enable LEDC IRAM safe APIs`.

## Использование

1. После подачи питания на дисплее появится выбранная форма сигнала и текущая частота (по умолчанию 1 кГц).
2. Вращение энкодера в обычном режиме листает формы сигналов, а генератор сразу переключает выход.
3. Нажмите на энкодер один раз, чтобы начать редактирование частоты. Строка частоты начнёт мигать.
4. Вращайте энкодер для изменения значения. Шаг автоматически подбирается в зависимости от диапазона (1, 5, 10 или 20 Гц).
5. Повторно нажмите кнопку энкодера для выхода из режима редактирования и фиксации частоты.

## Добавление собственных форм

- Таблицы отсчётов находятся в `src/waveform_tables.inc`. Вы можете сгенерировать новый массив из 128 значений (0…8191) и добавить его в файл.
- Зарегистрируйте новую форму в `src/waveforms.c`, указав имя и границы значений.
- После перезагрузки проекта новая форма появится в меню.

## Отладка и расширение

- Файл `src/main.c` содержит основную логику чтения энкодера, управления LEDC и обновления UI.
- Интерфейс отображения реализован в `src/ui.c`, а драйвер OLED — в `components/ssd1306` (без сторонних зависимостей).
- Для сложных фигур можно уменьшить длину таблицы или скорректировать частоту LEDC (`LEDC_BASE_FREQ`), учитывая ограничения ESP32-C6.
- При текущей частоте таймера LEDC (5 кГц) и 128 точках устройство гарантированно воспроизводит формы до ≈780 Гц (ограничение задаёт скорость прерываний). Для более высоких частот уменьшайте количество точек в таблицах и/или дорабатывайте логику обновления выборок.

## Решение проблем

- **На дисплее нет изображения** — проверьте адрес I²C (по умолчанию 0x3C) и правильность SDA/SCL. При необходимости измените адрес во `ui.c`.
- **Дребезг энкодера** — при необходимости добавьте аппаратные RC-цепочки или увеличьте программный дебаунс (см. функцию `read_button_pressed`).
- **Форма сигнала «зубчатая»** — увеличьте ёмкость в фильтре, либо используйте внешний ЦАП.
- **Сообщение «project_description.json не найден»** — возникает при запуске `idf.py` напрямую. Используйте команды PlatformIO (`pio run`, `pio run -t upload`, `pio run -t menuconfig`), либо перед запуском `idf.py` выполните `pio run`, чтобы сгенерировать необходимые артефакты.
- **`Could not use 'git describe'...` или ошибки `head-ref`** — начиная с версии прошивки 0.2.0 проект отключает проверку Git-метаданных ESP-IDF (переменная `IDF_SKIP_GIT_VERSION`). Если после обновления сообщение остаётся, удалите каталог `.pio/build` (`pio run -t clean`) и соберите проект повторно.
